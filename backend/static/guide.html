Agent Connect 使用指南

多智能体协作平台 — 输入自然语言任务，由 Planner 自动拆解 DAG，Research / Coder / Writer 等 Agent 并行协作，最终在右侧面板查看结果与 Token 用量。

快速开始

1

选择模板（可选）

在「运行」页顶部下拉框选择预设流水线，例如「调研+写作+翻译」；留空则由 Planner 自动规划。

2

输入任务并提交

在输入框描述你的目标，按 Enter 或点击「提交任务」。也可使用右侧「快速任务」一键填充示例。

3

跟踪进度与结果

右侧「任务时间线」实时更新；「任务结果」展示汇总、分步产出与 Token 用量；需要时可下载 artifact 文件。

三个主要页面

运行

提交任务、查看 DAG 拓扑与 Agent 消息流。适合日常使用与演示。

编排 (DAG)

可视化拖拽 Agent 节点、设置依赖与人工审批门，保存为自定义模板后提交。

管理

Admin 角色可创建 API Key，分配 viewer / operator / admin 权限给团队成员。

工作区路径（改现有项目 / 从零新建）

1

填写目录

在「运行」页「工作区路径」输入本机项目目录，例如 /Users/you/projects/my-app 或相对路径 ./my-new-service。留空则只在对话里生成代码，不写磁盘。

2

选择「编码开发」模板

Coder 会读取目录树与关键文件，并按 ```python path/to/file.py 格式写回文件；TestRunner 会在该目录根下执行 pytest -q .。

3

安全边界

路径必须在允许根目录内（默认：本仓库根 + 当前工作目录）。可在 .env 配置 WORKSPACE_ALLOWED_ROOTS。新建目录需 WORKSPACE_CREATE_IF_MISSING=true。

常用功能

  • API Key：右上角填写 Key 后，所有请求自动携带 X-API-Key 头。
  • 人工审批：审查未通过时，右侧会出现审批框，可选择通过、退回修改或终止。
  • 检查点 / 续跑：任务失败或完成后，可从检查点重放或从断点续跑。
  • Token 用量：任务完成后在「任务结果」上方显示累计 Token 与估算费用。
  • 快速任务：右侧预设按钮可一键填充常见场景（YOLO 服务、行业报告等）。

内置 Agent 职责

Planner

任务拆解与 DAG 调度

Research

调研、Arxiv/GitHub 工具检索

Coder

代码实现与 API 开发

Writer

文案、文档与报告撰写

Analyst

数据分析与竞品对比

Reviewer

质量审查与反馈

API 与开发

  • REST API 文档：/docs（OpenAPI Swagger）
  • 健康检查：GET /health
  • 提交任务：POST /tasks，body 含 input 与可选 template_id
  • 任务流：GET /tasks/{id}/stream（SSE 实时状态）
